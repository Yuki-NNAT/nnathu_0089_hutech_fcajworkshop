### Week 2 Achievements:

* Learned how to access AWS Billing and Cost Management to monitor account costs and the remaining AWS Credit.

* Created an AWS Budget with a suitable spending limit and configured email alerts for the selected cost thresholds.

* Understood that AWS Budgets data may not be updated immediately, so AWS costs should be monitored regularly.

* Understood the role of AWS IAM in managing identities and controlling access to AWS resources.

* Learned the purpose of the main IAM components:

  * **IAM User:** Represents a person or application that requires access to AWS.
  * **IAM Group:** Organizes multiple IAM users to simplify permission management.
  * **IAM Role:** Provides temporary permissions to users, applications, or AWS services.
  * **IAM Policy:** Defines which actions are allowed or denied for AWS resources.

* Understood the difference between the root user and an IAM user and why the root user should not be used for daily tasks.

* Learned the principle of least privilege, in which users and services receive only the permissions required to perform their tasks.

* Successfully enabled MFA, providing an additional layer of protection for the AWS account.

* Understood how AWS Support helps users resolve issues related to AWS accounts, billing, and services.

* Learned the process of creating and managing an AWS Support Request, including:

  * Selecting the support request type.
  * Selecting the related service or issue.
  * Describing the problem and providing the required information.
  * Selecting an appropriate contact method.
  * Monitoring the case status and providing additional responses.
  * Closing the request after the issue has been resolved.

* Understood the differences between account, billing, and technical support requests.

* Became more familiar with the AWS Management Console and completed basic tasks such as:

  * Searching for and accessing AWS services.
  * Reviewing resource configurations.
  * Monitoring the operating status of resources.
  * Performing basic resource management tasks.
  * Accessing cost management and account security settings.

* Completed introductory AWS lectures and several basic AWS labs using videos and online documentation.

* Recorded the steps, results, issues, and solutions encountered during the labs.

* Completed the objectives for Week 2 and gained the foundational AWS knowledge required to begin researching and selecting a project idea in the following week.
